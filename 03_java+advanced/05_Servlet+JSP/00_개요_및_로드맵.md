- Spring Boot의 배경 기술인 Servlet/JSP와 WAS(Tomcat)의 동작 원리를 이해

## 📋 빠른 참조

- WAS: 동적 웹 콘텐츠 실행(서블릿/JSP) — Tomcat 10.1.x 기준
- Web Server: 정적 파일 전달(Nginx/Apache)
- Servlet: 자바로 작성하는 컨트롤러 컴포넌트
- JSP: HTML 템플릿에 동적 값을 삽입하는 뷰
- 포트 기본값: http://localhost:8080/

## 왜 지금 배우나? (비유)

- Web Server는 “정적 자료를 잘 건네주는 우체국”, WAS는 “편지를 읽고 새로 작성해 보내는 작가” 역할.
- Spring Boot는 ‘작가 도구 세트(Framework)’로, 그 밑바탕인 Servlet/Tomcat을 이해하면 다른 런타임/배포 환경에도 빠르게 적응 가능.
